1 Mappen structuur

  • “routes.php” en “index.php” zijn de enige bestanden waar de statements “require_once” en “include” gebruikt mogen worden.
  • Alle bestanden hebben de extentie “.php”, behalve bestanden waarin CSS code staat, deze hebben de extentie “.CSS”.
  • CSS mag alleen in het bestand “routes.php” geïncludeerd worden.
  • De mappen classes, libraries en miscellaneous vallen onder de term ondersteuning.
  • De mappen views, forms en navigation vallen onder de term controllers.
  • De mappen classes, libraries en miscellaneous zijn ter ondersteuning voor de bestanden die in de mappen views, forms, naviation zitten.
  • De logica van de applicatie vindt plaats in de controllers.

Alle bestanden in deze applicatie hebben gemeen dat ze dezelfde extensie delen, namelijk “.php”. De rede hiervoor is dat als je in een “.php” bestand (waar voor het voorbeeld alleen HTML code in staat) PHP code wilt toevoegen, meteen de daarbij horende “color scheme” zichtbaar wordt.

Hieronder zie je de mappenstructuur van de website https://t04-er-1718.clow.nl/. Bestanden zijn cursief weergeven en mappen niet. De map “/domains/t04-er-1718.clow.nl/e_register” is ingedeeld in een aantal categorieën. Deze categorieën staan in de volgende subhoofdstukken beschreven.

  • /domains/t04-er-1718.clow.nl/e_register
    • /api
    • /classes
    • /libraries
    • /miscellaneous
    • /views
    • /forms
    • /navigation
    • /routes.php
  • /domains/ t04-er-1718.clow.nl/public_html
    • /media
      • /profile
      • /logo
    • /index.php

1.1 Mappen

In de onderstaande subhoofdstukken staat het doel van de mappen beschreven.

1.1.1 Api

  • In deze map komt alles te staan wat met de api te maken heeft.
  • Het is verstandig om uit deze map te blijven.

1.1.2 Classes

  • In deze map komen alle classes te staan.
  • Elke class komt in zijn eigen bestand te staan.
  • Het is alleen toegestaan een class te definiëren in zo’n bestand. Dus GEEN andere programmatuur in deze bestanden!
  • De naam van de class is hetzelfde als de naam van het bestand waar het in staat.
  • Gebruik “PascalCased” voor class namen, “camelCased” voor methods en properties.

1.1.3 Libraries

  • In deze map komen alle libraries te staan.
  • Het is alleen toegestaan een functie te definiëren in zo’n bestand. Dus GEEN andere programmatuur in deze bestanden!
  • De naam van de library moet een passende alles omvattende korte naam zijn.

1.1.4 Miscellaneous

  • In deze map komt alles te staan wat niet onder een van de categorieën valt.
  • Overleg wel eerst met de F. Dijkstra of dit nodig is.

1.1.5 Views

  • In deze map komt alles te staan wat data aan de gebruiker laat zien.
  • Elke tabel, grafiek, array, etc moet in een apart bestand.

1.1.6 Forms

  • In deze map komen alle forms te staan.
  • Elke form moet in een apart bestand.
  • De verwerking van de gegevens, afkomstig van de form, moeten in datzelfde bestand verwerkt worden.

1.1.7 Navigation

  • In deze map komen onder andere headers, footers, navigatiebalken, etc te staan.
  • Elk element moet in een apart bestand.

3 Naamconventies

Een naam is een benoeming van een class, method, propertie, fuction, variable, constant, folder, file en class file. Voor alle namen moet de taal Engels gebruikt worden. De namen moeten kort en bondig zijn. Namen mogen niet dubbel gebruikt worden. Binnen een functie hoef je niet rekening te houden met namen die al gebruikt zijn buiten die functie.

3.1 Overzicht

In deze tabel staan alle soorten namen die door een programmeur gedefinieerd kan worden. Daarachter staat welk format gebruikt moet worden voor de naamgeving. In de verklarende woordenlijst kun je de definities vinden van deze formats.

Classes PascalCased
Methods camelCased
Properties camelCased
Functions lower_case
Variables lower_case
Constants UPPER_CASE
Folders lower_case
Files lower_case
Class files PascalCased

3.2 Gereserveerde namen

Alle namen van folders, files en class files die in de mappenstructuur van hoofdstuk 1 staan zijn gereserveerde namen en mogen dus niet gebruikt worden voor andere folders, files en class files.

Soort Naam Omschrijving
Variabele $i Wordt gebruikt als counter value in for loops
Variabele $j Wordt gebruikt als counter value in nested for loops
Variabele $k Wordt gebruikt als counter value in nested for loops
Variabele $APP_PATH Root van je applicatie

4.3 Naamconventies

Beginnend met een hoofdletter, spaties en de daarop volgende letter vervangen door de corresponderende hoofdletter.

4.3.1 PascalCased

Beginnend met een hoofdletter, spaties en de daarop volgende letter vervangen door de corresponderende hoofdletter.

Voorbeelden
  • PestJSON
  • APIVerbinding
  • Pest

4.3.2 camelCased

Beginnend met een kleine letter (of 2 underscores), spaties en de daarop volgende letter vervangen door de corresponderende hoofdletter.

Voorbeelden
  • __construct
  • prepRequest

4.3.3 lower_case

Alleen kleine letters, spaties vervangen voor een underscore.

Voorbeelden
  • show_login_form
  • validate
  • create_account

4.3.4 UPPER_CASE

Alleen hoofdletters, spaties vervangen voor een underscore.

Voorbeelden
  • APP_PATH
  • PI
